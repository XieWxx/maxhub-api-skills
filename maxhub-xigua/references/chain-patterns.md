# 西瓜视频数据采集与分析链式调用模式库

## 核心原则

1. **按需调用**：用户需求能用1个API解决就不要调用2个
2. **参数传递**：链式调用中，上一步的返回值作为下一步的参数
3. **数量控制**：批量获取详情时默认最多10条，需用户确认才能更多
4. **错误中断**：任何一步失败则停止，向用户说明原因

---

## 已知场景模式

### 模式1：搜索→视频详情

**触发场景**：用户搜索某个主题/关键词后想看具体视频内容

**调用步骤**：

1. `app/v2/search_video` — 搜索关键词获取视频列表和ID（参数：keyword）
2. `app/v2/fetch_one_video` — 用视频ID获取完整详情（参数：ID（从上一步获取））

**示例**：
```
用户: "搜索关于xxx的视频"
→ Step1: app/v2/search_video?keyword=xxx
→ Step2: app/v2/fetch_one_video?id=<Step1返回的ID>
```

---

### 模式2：视频→评论

**触发场景**：用户想看视频的评论

**调用步骤**：

1. `app/v2/fetch_one_video` — 获取视频基本信息（参数：item_id）
2. `app/v2/fetch_video_comment_list` — 获取评论列表（参数：视频ID）

**示例**：
```
用户: "这个视频的评论"
→ Step1: app/v2/fetch_one_video?id=xxx
→ Step2: app/v2/fetch_video_comment_list?id=xxx
```

---

### 模式3：搜索→视频详情→评论

**触发场景**：用户搜索后想看视频详情和评论

**调用步骤**：

1. `app/v2/search_video` — 搜索关键词获取视频列表（参数：keyword）
2. `app/v2/fetch_one_video` — 获取视频详情（参数：ID（从搜索结果获取））
3. `app/v2/fetch_video_comment_list` — 获取评论（参数：视频ID）

**示例**：
```
用户: "搜索xxx并看评论"
→ Step1: app/v2/search_video?keyword=xxx
→ Step2: app/v2/fetch_one_video?id=<Step1返回>
→ Step3: app/v2/fetch_video_comment_list?id=<同上>
```

---

### 模式4：创作者信息→视频列表

**触发场景**：用户想查看某个创作者的视频

**调用步骤**：

1. `app/v2/fetch_user_info` — 获取创作者信息（参数：user_id）
2. `app/v2/fetch_user_post_list` — 获取创作者视频列表（参数：user_id）

**示例**：
```
用户: "查看这个创作者的视频"
→ Step1: app/v2/fetch_user_info?user_id=xxx
→ Step2: app/v2/fetch_user_post_list?user_id=xxx
```

## 未知场景处理策略

当用户需求不在上述已知模式中时，按以下策略处理：

### 单API直接调用

**触发场景**：用户需求明确，单个API即可满足

**处理策略**：直接匹配最合适的API调用，无需链式

### 先搜索后详情

**触发场景**：用户提到某个视频或创作者但不知道具体ID

**处理策略**：先用搜索类API获取ID，再用详情API获取完整数据

### 先列表后逐条

**触发场景**：用户需要某个创作者的视频中每条的详细数据

**处理策略**：先获取创作者视频列表，再对每条视频调用详情API（注意控制数量，默认最多10条）

### 多维度组合分析

**触发场景**：用户需求跨多个数据维度（如同时需要用户数据、内容数据、互动数据）

**处理策略**：按维度分别调用对应API，将结果组合输出分析报告

---

## 链式调用决策树

```
用户需求
  │
  ├─ 是否知道具体ID？
  │    ├─ 是 → 直接调用详情API
  │    └─ 否 → 先搜索获取ID
  ├─ 是否需要多个数据维度？
  │    ├─ 是 → 按维度分别调用API，组合输出
  │    └─ 否 → 单API直接返回
  ├─ 是否需要批量数据？
  │    ├─ 是 → 先获取列表，再逐条获取详情（默认≤10条）
  │    └─ 否 → 单条获取
  └─ 场景不明确？
       └─ 先用搜索API探索，再根据结果决定下一步
```
